# Анализ решений для составления расписаний (кратко)

## 1) Что обычно есть в готовых продуктах
- Автогенерация расписания по набору правил и ограничений.
- Ручные правки с фиксацией отдельных занятий.
- Поддержка потоков, подгрупп и нескольких смен.
- Ежедневные замены преподавателя/аудитории.
- Экспорт в Excel/PDF/веб.

## 2) Что встречается у распространенных решений
- `Экспресс-расписание ВУЗ`: сильная сторона - быстрое досоставление и режим замен; слабая - ограниченная гибкость бизнес-логики под уникальные правила конкретного ВУЗа.
- Набор локальных модулей в 1С/ERP: сильная сторона - интеграция с кадровыми и учебными данными; слабая - сложность кастомизации UI и алгоритмов без доработки.
- Самописные Excel-макросы: сильная сторона - быстрый старт; слабая - отсутствие строгой проверки конфликтов и масштабируемости.

## 3) Вывод для этого проекта
- Нужна стратегия `constraint + optimization` (CP-SAT), а не только шаблонные правила.
- Критично иметь отдельные экраны:
  - занятость аудиторий,
  - расписание ППС,
  - окно замен и фиксации слотов.
- Данные должны храниться в переносимом формате (JSON), чтобы быстро адаптировать учебные планы.

## 4) Чем текущий MVP отличается
- Учитывает жесткие ограничения по ППС, аудиториям, группам и периодам в неделях.
- Поддерживает потоки/подгруппы, смены групп, фиксированные аудитории и блокировки по слотам.
- Поддерживает быстрые замены аудитории/ППС и повторную генерацию.
